Module 1 — Foundation

What is Hugging Face — and why does it matter for sourcing?

A 3-minute primer before you run a single search.

Think of it as
Hugging Face (HF) is the GitHub of AI. Just like developers post code to GitHub, AI practitioners publish their models, apps, and datasets to HF — publicly, with their username attached. Most people in the AI builder space have a profile there, even if they don't know they're "being sourced."
Models
AI models people trained or fine-tuned. Like publishing a book.
Spaces
Live demos and apps. Someone built and shipped something. Highest signal.
Datasets
Training data someone curated. Rare skill — very high value.
Who are you looking for?
The AI Builder persona — someone at the intersection of software engineering and LLMs. Not a researcher who just reads papers. Not a data scientist who only runs notebooks. Someone who builds agentic systems — autonomous AI that can take actions, use tools, and chain reasoning steps.
LangGraph / LangChain CrewAI / AutoGen Fine-tuning / LoRA / RLHF RAG / vector stores Function calling / MCP vLLM / TGI / deployment
Good news: You do NOT need a Hugging Face account. All profiles, Models, Spaces, and Datasets are publicly viewable. Google has already indexed them. Your job is to use the right search strings to surface them.

Module 2 — Core skill

Boolean strings that actually work

Copy, paste, run in Google. No HF login needed.

How it works The site: operator
Google has already crawled HF. site:huggingface.co/spaces tells Google to search only inside HF Spaces. You combine that with skill keywords to find builders.

How to read Google results for HF profiles
1
Look at the URL
Results with huggingface.co/{username}/spaces/{name} or huggingface.co/{username}/ — that's a person's page, not a documentation page.
2
Extract the username
The part after huggingface.co/ and before the next slash is the username. Navigate directly to huggingface.co/{username} to see their full profile.
3
Exclude docs + blog
Add -site:huggingface.co/docs -site:huggingface.co/blog to your string to remove documentation pages from results.

Module 5 — Verify

Cross-verify on other platforms

HF gives you the username. These platforms give you the person.

The identity ladder: HF username → GitHub (linked in bio) → GitHub README (often has LinkedIn/Twitter) → LinkedIn. Most builders can be fully triangulated in under 3 minutes.
GitHub
Strongest signal. Check contribution streak, repo stars, Python dominance, topic tags on repos.
site:github.com "[username]" "langchain" OR "transformers"
LinkedIn
Ground truth for location + employer. Use name + org from HF profile to find them.
site:linkedin.com/in "[full name]" "LangChain" OR "LLM"
Stack Overflow
High-rep users in ml / pytorch / langchain tags = confirmed technical depth.
site:stackoverflow.com/users "[name]"
Kaggle
Grandmaster/Master tier = elite ML. Notebook count signals teaching contribution.
site:kaggle.com "[username]" OR "[full name]"
Google Scholar
Confirms researcher identity. Look for applied AI papers with linked code repos.
site:scholar.google.com "[full name]" "agentic"
Location inference
HF has no structured location. Infer via GitHub profile field or LinkedIn — never filter in the HF string itself.
site:github.com "[username]" "San Francisco" OR "New York"

Module 6 — Rules + Process

Compliance guardrails + your sourcing SOP

The rules that keep sourcing clean — and the workflow to follow every session.

⚠ EEOC + JPMC sourcing compliance — read this first
Do NOT filter by location as a proxy for national origin or citizenship at the sourcing stage. If a role requires US work authorization, that conversation happens in screening — not in your Boolean string. Source broadly on skill signals. Qualify on eligibility during the outreach call. Never record inferred demographics (ethnicity, age, gender) in your CRM from profile photos or names. When in doubt, check with your TA legal partner.
Your 6-step sourcing SOP
1
Run 3–4 Google boolean strings
Use the string builder or copy from the module 2 presets. Target Spaces first (highest signal), then Models. Scan first 2 pages of results each.
Batch mode: run all strings in one session before qualifying any. Don't context-switch — it kills velocity.
2
Extract usernames from URLs
The path segment after huggingface.co/ is the username. Navigate to huggingface.co/{username} — public, no login needed. Add username to tracking sheet.
3
Qualify the profile (60-second rule)
Check: Spaces published (3+ = strong), model downloads (500+ = traction), GitHub linked (yes/no), last activity (within 90 days?). Score high/mid/skip.
4
Pivot to GitHub
Click GitHub link from HF bio. Confirm: Python-heavy repos, recent commits, stars on agent or LLM repos. This also often surfaces their real name.
5
Confirm identity on LinkedIn
Search real name + org from HF profile on LinkedIn. Confirm current employer, seniority, and location. Only add to pipeline once confirmed.
Location and work auth screening happens here — in conversation, not in your search string.
6
Personalize your outreach
Reference something specific: "I saw your [SpaceName] agent on HF — the approach to [specific thing] stood out." Generic InMails get ignored. Specific ones get replies.
